# Skills

本目录存放可复用的任务操作规则，用于在特定场景中稳定触发检查、约束和执行步骤。

这些规则面向反复出现的任务类型，例如代码验证、副作用安全、交付边界、状态汇报、故障排查、文档审查和工程治理。它们把经验判断沉淀为可复用流程，减少遗漏、误判和过度自信。

每个目录对应一个独立主题，通常包含：

```text
<skill-name>/
└── SKILL.md
```

`SKILL.md` 描述该主题的适用场景、核心原则、检查步骤和跳过条件。不同主题之间保持独立，便于单独维护、组合使用和持续扩展。

这些文件关注实际执行质量：什么时候需要多验证一步，什么时候需要阻断风险，什么时候需要清楚说明边界，什么时候可以停止继续加工。

整体目标是让复杂任务的执行过程更稳、更清楚、更可复用。
